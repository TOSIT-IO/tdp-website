# Installation de l'environnement TDP

Cette page permet de créer l'environnement nécéssaire au déploiement de TDP. Il pourra à la fois être utilisé pour déployer TDP manuellement ou avec [TDP Manager](#).

Une fois configuré, l'environnement doit avoir la structure suivante :

```txt
tdp/
├── ansible_dependencies/
│   └── ansible_collections/
│       ├── ansible/
│       ├── community/
│       ├── tosit/
│       │  ├── tdp/
│       │  ├── tdp_extra/
│       │  ├── tdp_observability/
│       │  └── tdp_prerequisites/
│       └── *.info/ # Fichiers de métadonnées des collections
├── inventory/
│   ├── group_vars/
│   │   └── all.yml
│   ├── hosts
│   ├── topologies/
│   │   ├── 01_tdp
│   │   ├── tdp_extra
│   │   ├── tdp_observability
│   │   └── tdp_prerequisites
│   └── tdp_vars.yml
├── ansible.cfg
└── requirements.yml
```

## Prérequis

Le seul prérequis pour installer l'environnement TDP est un système d'exploitation sur lequel [Ansible](https://www.ansible.com/) est installé. Les systèmes d'exploitation suivants ont été testés :

- CentOS 7
- Rocky 8
- AlmaLinux 8

L'installation peut être effectuée sur d'autres systèmes d'exploitation, mais il est possible que certaines fonctionnalités ne soient pas disponibles.

Note : Dans le cadre de l'installation de TDP Manager, il est recommandé d'utiliser directement la version d'Ansible contenue dans l'[envionnement virtuel de TDP Manager](#).

Les instructions fournies sur cette page sont à réaliser dans le même répertoire. Par exemple, un répertoire `tdp` à la racine de votre utilisateur :

```bash
mkdir ~/tdp
cd  ~/tdp
```

## Configuration d'Ansible (`ansible.cfg`)

TDP est basé sur Ansible. Un fichier de configuration, `ansible.cfg`, est nécessaire pour définir des paramètres nécéssaires au projet. Il précise en partulier les chemins vers les collections et les rôles.

Créer le fichier `ansible.cfg` à la racine du répertoire avec le contenu suivant :

```ini
[defaults]
inventory=inventory ; OBLIGATOIRE permet de définir le répertoire de l'inventaire Ansible.
display_skipped_hosts=False ; RECOMMANDE permet de ne pas afficher les tâches Ansible qui sont _skipped_ pour éviter d'encombrer les logs.
collections_paths=ansible_dependencies ; RECOMMANDE permet de définir les chemins pour les collections. Le premier chemin de cette liste sera celui utilisé pour les dépendances téléchargées avec `ansible-galaxy`.

; RECOMMANDE permet d'accélérer considérablement le lancement des playbooks, car l'installation de TDP implique de nombreux appels à la commande `ansible-playbook`.
[inventory]
cache = true
cache_plugin = jsonfile
cache_timeout = 7200
cache_connection = .cache

; OBLIGATOIRE permet d'activer le plugin `tdp_vars`, qui construit les variables utilisées par les collections Ansible de TDP.
[tdp]
vars = tdp_vars

; OBLIGATOIRE permet d'activer le passage à l'utilisateur `root` pour toutes les tâches Ansible.
[privilege_escalation]
become=True
become_user=root
```

## Installation des collections (`requirements.yml` et `ansible_dependencies/`)

Le projet s'articule autour d'un ensemble de collections permettant de déployer des composants. Pour en savoir plus sur les collections disponibles ainsi que leur contenu, consulter la page [Collections](#).

Les collections à installer sont à déclarer dans le fichier `requirements.yml`.

Créer le fichier `requirements.yml` à la racine du répertoire en ajoutant les collections TDP souhaitées:

```yml
---
collections:
  # Dépendances obligatoires
  - name: ansible.posix
    version: 1.5.4
  - name: community.crypto
    version: 2.14.0
  - name: community.general
    version: 7.1.0
  - name: community.grafana
    version: 1.5.4
  - name: community.postgresql
    version: 2.4.2

  # Obligatoire
  # Contient les services principaux (HDFS, YARN, Hive, etc.) ainsi que les plugins nécessaires à la gestion des variables.
  - name: https://github.com/TOSIT-IO/tdp-collection
    type: git
    version: master
  # Optionnelle
  # Contient des composants supplémentaires (Kafka, Hue, Airflow, etc.).
  - name: https://github.com/TOSIT-IO/tdp-collection-extras
    type: git
    version: master
  # Optionnelle
  # Contient des composants dédiés à l'observabilité (Prometheus, Grafana, etc.).
  - name: https://github.com/TOSIT-IO/tdp-observability
    type: git
    version: main
  # Optionnelle
  # Contient des rôles permettant d'installer les prérequis sur des machines (CentOS 7 ou Rocky 8).
  - name: https://github.com/TOSIT-IO/tdp-collection-prerequisites
    type: git
    version: master
```

Exécuter la commande suivante pour télécharger les collections et leurs dépendances :

```bash
ansible-galaxy install -r requirements.yml
```

Les dépendances seront installées dans le dossier `ansible_dependencies/ansible_collections` (spécifié par la propriété `default.collections_paths` du fichier `ansible.cfg`).

## Définition de l'inventaire Ansible (`inventory/`)

Créer manuellement la structure de l'inventaire Ansible :

```bash
mkdir -p inventory/{group_vars,topologies}
```

Installer le plugin `tdp_vars` :

```bash
echo "plugin: tosit.tdp.tdp_vars" > inventory/tdp_vars.yml
```

### Variables globales (`inventory/group_vars/all.yml`)

Certaines variables **doivent** être communes à tous les hôtes. Elles sont à spécifier dans le fichier `inventory/group_vars/all.yml`.

Créer le fichier `inventory/group_vars/all.yml` en complètant les valeurs suivantes :

```yml
---
domain: # Domaine DNS du cluster
realm: # Nom du royaume Kerberos
kerberos_admin_principal: # Identifiant de l'administrateur Kerberos
kerberos_admin_password: # Mot de passe de l'administrateur Kerberos
kadmin_principal: '{{ kerberos_admin_principal }}' # Identifiant de l'administrateur Kerberos
kadmin_password: '{{ kerberos_admin_password }}' # Mot de passe de l'administrateur Kerberos
```

**Par exemple** :

```yml
---
domain: tdp
realm: TDP.LOCAL
kerberos_admin_principal: 'admin/admin@{{ realm }}'
kerberos_admin_password: admin123
kadmin_principal: '{{ kerberos_admin_principal }}'
kadmin_password: '{{ kerberos_admin_password }}'
```

### Hôtes (`inventory/hosts`)

Le fichier `inventory/hosts` permet de définir les hôtes sur lesquels déployer TDP. C'est un fichier d'[inventaire Ansible](https://docs.ansible.com/ansible/latest/inventory_guide/intro_inventory.html).

Note : Dans le contexte de TDP Manager, il est possible de tester les fonctionalités du Manager sans définir d'hôtes. Dans ce cas, les commandes Ansible ne seront pas exécutées et seront considérées comme réussies. Pour activer ce mode, ajouter la variable `export TDP_MOCK_DEPLOY=True` dans le fichier `.env` à la racine du répertoire.

Le fichier d'inventaire doit définir au minimum les groupes suivants :

- `edge`
- `master`
- `master1`
- `master2`
- `master3`
- `worker`

Créer un fichier `inventory/hosts` adapté à votre infrastructure en complétant le contenu suivant :

```ini
[edge]
; Liste des hôtes du groupe edge

[master1]
; Liste des hôtes du groupe master1

[master2]
; Liste des hôtes du groupe master2

[master3]
; Liste des hôtes du groupe master3

[master:children]
master1
master2
master3

[worker]
; Liste des hôtes du groupe worker
```

**Par exemple**, pour un cluster de 6 machines (1 edge, 3 masters et 2 workers), le fichier `inventory/hosts` sera le suivant :

```ini
[edge]
edge-01 edge01.tdp.local

[master1]
master-01 master01.tdp.local

[master2]
master-02 master02.tdp.local

[master3]
master-03 master03.tdp.local

[master:children]
master1
master2
master3

[worker]
worker-01 worker01.tdp.local
worker-02 worker02.tdp.local
```

### Topologies (`inventory/topologies/`)

Les fichiers de topologies permettent de définir l'organisation des composants (HDFS NameNode, Hive Metastore, etc.) sur les différents hôtes. Ce sont également des fichiers d'inventaire Ansible.

Chaque collection est fournie avec un fichier de topologie d'exemple (fichier `topology.ini`).

`tdp-collection` est la collection principale. Son fichier de topologie doit être lu en premier. Pour cela, il est conseillé de préfixer son nom par `01_`.

Copier le fichier de topologie de `tdp-collection` dans le répertoire `inventory/topologies` :

```bash
cp ansible_dependencies/ansible_collections/tosit/tdp/topology.ini inventory/topologies/01_tdp
```

Si d'autres collections sont installées, il faut également ajouter leurs topologies afin qu'elles soient prise en compte. Les topologies des autres collections peuvent s'appuyer sur les noms des groupes définis dans la topologie de `tdp-collection`.

```bash
cp ansible_dependencies/ansible_collections/tosit/tdp_extra/topology.ini inventory/topologies/tdp_extra
cp ansible_dependencies/ansible_collections/tosit/tdp_observability/topology.ini inventory/topologies/tdp_observability
cp ansible_dependencies/ansible_collections/tosit/tdp_prerequisites/topology.ini inventory/topologies/tdp_prerequisites
```

Ces fichiers sont à éditer en fonction de la répartition des composants souhaitée. Pour désactiver un composant, il suffit de le retirer des topologies.

**Par exemple**, la topologie suivante permet de déployer un Hive Metastore sur les hôtes des groupes `master2` et `master3` :

```ini
[hive_ms:children]
master2
master3
```
