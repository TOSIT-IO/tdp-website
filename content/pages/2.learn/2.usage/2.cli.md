# Utilisation de la CLI d'administration

`tdp-lib` est un SDK qui expose des objets qui permettent de planifier, exécuter (via [Ansible](https://www.ansible.com/)) et monitorer des déploiements basés sur un [DAG](https://fr.wikipedia.org/wiki/Graphe_orient%C3%A9_acyclique).

`tdp-lib` embarque une **CLI** destinée aux administrateurs.

## Prérequis

Consulter la documentation d'[intallation de TDP Manager]() afin de pouvoir utiliser la CLI d'administration.

### Activation de l'environnement virtuel

Activer l'environnement virtuel éventuellement créé lors de l'installation de TDP Manager:

```bash
# Activation de l'environnement virtuel (facultatif)
source .venv/bin/activate
# Affiche l'aide
tdp --help
```

Une fois l'environnement virtuel activé, la CLI peut être utilisée via la commande `tdp`.

## Premier déploiement

Un déploiement s'effectue en deux étapes : la planification et l'exécution. Les commandes suivantes permettent de déployer un cluster TDP en suivant le DAG dans son intégralité.

```bash
# Planification du déploiement
tdp plan dag
# Exécution du déploiement
tdp deploy
```

## Mise à jour d'une configuration

Lors du changement d'une configuration, il est nécéssaire de reconfigurer l'ensemble des services impactés. Par exemple, lors du changement de la configuration de HDFS, il est nécéssaire de reconfigurer également les composants de HBase qui utilisent HDFS comme stockage.

```bash
# Edition de la configuration des datanodes'
tdp vars edit hdfs datanode
# Planification du déploiement
tdp plan reconfigure
# Exécution du déploiement
tdp deploy
```

## Autres commandes

De nombreuses autres commandes sont disponibles. Pour plus d'informations, consulter la documentation de la CLI d'administration.

```bash
# Affiche l'aide
tdp --help
```
